# WeRSS: 从信息感知到知识萃取 (Idea & Manifesto)

## 1. 时代背景：被围墙困住的“核心资产”
在中文互联网生态中，微信公众号是高质量原创内容、深度行业洞察、前沿科技解读最集中的阵地。然而，这个生态是极其**封闭**的：
- **不可检索**：公网搜索引擎无法触达，知识无法被有效沉淀和二次挖掘。
- **不可订阅**：缺乏开放的 RSS/API 接口，用户被迫困在手机端的瀑布流中。
- **信噪比极低**：在海量更新中，90% 的内容是营销、重复或低质的，真正的“知识金块”被埋没。

## 2. WeRSS 的定位：知识漏斗的最宽处 (The Sensory Layer)
WeRSS 不仅仅是一个爬虫或 RSS 生成器。它是整个知识治理体系的**前哨感知系统**。

如果把知识比作黄金，那么：
- **互联网/公众号** 是原始矿场。
- **WeRSS** 是自动化的**矿石筛选机**（负责发现、初筛、结构化）。
- **Knoss** 是**炼金实验室**（负责实体提炼、标准化、图谱构建）。

**核心哲学：Curation > Collection (提炼大于采集)**
我们不追求收集所有信息，我们追求在信息爆炸中通过“热度分析”和“语义标签”快速定位最有价值的 1% 核心内容。

## 3. 三大技术支柱
- **多维感知器 (Sensors)**：通过 Web/App/API 多种驱动，实现低侵入、全自动的抓取，将封闭的公众号内容转化为可流动的结构化数据。
- **动态过滤器 (Filters & Heat Tracking)**：
  - **热度追踪**：通过互动数据识别真正具有社会价值和专业深度的内容。
  - **AI 标签系统**：基于 GLiNER 或 LLM 进行轻量化主题提取，实现“内容驱动”而非“账号驱动”的分类。
- **知识桥梁 (Bridges)**：
  - **面向人类**：输出优雅的 RSS、Markdown 或 PDF，回归纯粹的深度阅读体验。
  - **面向 AI (RAG Ready)**：为后续的 Knoss 系统提供清洗后的、带有元数据的干净语料，解决 RAG 系统的信噪比难题。

## 4. 协同效应：WeRSS + 知识治理系统
WeRSS 解决了知识治理的**“源头活水”**问题：
1. **WeRSS 发现热点**：自动捕获某个垂直领域（如“前沿技术”）的最新讨论。
2. **WeRSS 初筛去噪**：剔除营销软文，保留高价值深度长文。
3. **知识系统深度加工**：从文章中提取核心概念、实体关系，并由专家审核入库。
4. **最终产出**：一个具备**实时性**（WeRSS 功劳）和**严谨性**（后端系统功劳）的私有知识图谱。

## 5. 未来展望：Agentic Knowledge Feed
WeRSS 的终极目标是进化为一个**自主的知识代理 (Knowledge Agent)**：
它能理解用户的研究领域，主动在全网感知波动，自动完成“发现-初筛-总结-入库”的全链路逻辑，让用户从“主动搜寻信息”转变为“坐享经过治理的深度洞察”。
