# 路线图与待办（Roadmap / TODO）

README 中有本页内容的**摘要表**；完整待办与勾选仍以本文为准。完成项可改为 `[x]` 或移到「已完成」小节。

面向教学、研究与社区协作的跟踪清单。

## 1. 微信公众号文章生成 Skill（按主题成文）

**背景：** 由学生/课题组成员扩展的专项能力：根据给定 **topic（主题）** 生成适合微信公众号发布的文章草稿或结构化大纲。

| 状态 | 待办项 |
|------|--------|
| [ ] | 明确输入输出契约：topic、受众、字数、是否带摘要/标签、是否对接现有文章入库流程 |
| [ ] | 与现有采集/编辑流水线对齐：是否写入同一数据库、是否复用 `Article` 模型与审核状态 |
| [ ] | 安全与合规：生成内容免责声明、人工审核节点、敏感词/事实核查策略（课程项目可文档化即可） |
| [ ] | 将 Skill 封装为可调用接口（内部 API 或脚本），并在 README 或 `docs/` 中补充使用说明 |
| [ ] | 准备 1～2 个可复现的示例 topic 与预期输出，便于验收与答辩 |

**建议标签（Issue）：** `skill`、`content-generation`、`wechat`

## 2. 关键词提取：轻量化 GLiNER 方向（自研）

**背景：** 团队在探索比现有方案更 **轻量** 的抽取方案，基于 **GLiNER** 类模型做关键词 / 实体级信号，与当前标签管线（如 KeyBERT、LLM）形成对照或级联。

| 状态 | 待办项 |
|------|--------|
| [ ] | 调研 GLiNER 与中文/多语场景下的适用边界、推理成本与依赖体积 |
| [ ] | 确定评测集：与现有 `ARTICLE_TAG_*` 行为对齐的离线指标（F1、人工抽检协议） |
| [ ] | 实现可选后端：`ARTICLE_TAG_EXTRACT_METHOD=gliner`（或独立开关），失败时回退到现有方法 |
| [ ] | 文档化：模型来源、许可证、是否在默认镜像中打包或按需下载 |
| [ ] | （可选）整理短论文 / 技术报告素材，便于学术合作者复现 |

**建议标签（Issue）：** `research`、`tags`、`gliner`、`performance`
